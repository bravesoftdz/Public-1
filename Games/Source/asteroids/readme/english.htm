The Game

After launching Asteroids.exe, the main menu appears. Select NEW GAME to play. You can always come back with ESC.

The player starts with his / her spaceship in a level with two big asteroids. You can turn and accelerate it using the cursor keys, and fire shots using the space bar. You can change these keys if you want to.

With every single shot, the ship's cannon gets heated by 250° degree centigrade. If the cannon's temperature is higher than 1000°, no shots can be fired. The cannon permanently cools down by one hundredth of it's temperature - it needs more than one second from 1° to 0°, but only 0,2 seconds from 1250° to 1000°.

If the spaceship is hit by an asteroid, the player looses 512 points and a new spaceship is created. The spaceship is after a collision (or after a level change) invincible for ca. 3 seconds, and is blinking. It's also in the center of the screen, and it's speed is reduced to zero.

If a shot or the spaceship hits an asteroid, the asteroid gets split. In a new level there're only asteroids of the 1st generation, and with each splitting two (smaller) asteroids of the next generation are created. Finally, the 4th generation vanishes completely without creating new asteroids. For each splitted asteroid, the player gets 12 * asteroid generation points. Additionally, each shot costs 12 points.

The result looks like this:

generation points per asteroid count total points shot loss result
1st 1 * 12 = 12 1 12 * 1 = 012 1 * 12 = 12 000
2nd 2 * 12 = 24 2 24 * 2 = 048 2 * 12 = 24 024
3rd 3 * 12 = 36 4 36 * 4 = 144 4 * 12 = 48 096
4th 4 * 12 = 48 8 48 * 8 = 384 8 * 12 = 96 288
total points for one big asteroid: 408


When all asteroids are destroyed, the next level begins. There you get at least two big asteroids. If the player's score is positive, then score / 512 asteroids are added. Example: If you have 5120 points, you get 12 big asteroids in the next level.

The Final Aim

Basically, you can play forever. The final aim is to get the highest score, which can only be achieved by destroying many asteroids while not being destroyed by an asteroid. That means that only the best will be on top of the list.

The game can always be stopped by selecting STOP GAME in the main menu. But if you select QUIT (or press Alt+F4), the program quits immediately and an eventual highscore is ignored.

The highscore list is not heavily protected because I publish the program's source code.

Options




You can always access the options by selecting OPTIONS in the main menu. The first seven settings can also altered by using the function keys F1 to F7.

  (F1) stop game if it looses focus The game pauses if another application becomes active.
  (F2) first person shooter mode The player is always in the middle.
  (F3) beep if shots launch and hit The internal speaker gets active if shots are fired or asteroids hit.
The resulting tones can differ on different computers.
  (F4) limit frame rate to 100 FPS The game is slowed down when the frame rate gets higher than 100 FPS. That's especially useful on systems with fluctuating frame rates.
  (F5) accelerate if FPS < 100 On slower systems, the frame rate can get much lower than 100 FPS. This setting compensates by accelerating all movements of the game.
This setting is not recommended, because accelerated movements can reduce the "reality" of the game. It could happen that shots go through asteroids, because they simply "jump over" that particular part of the screen.
  (F6) use partial 2D polygons A polygon that gets near to the screen border is normally not displayed on the other side of the screen. But if this box is checked, each polygon is displayd nine times, and the visible screen is in the center of a 3x3 field of screens.
This setting can heavily slow down the program, because it's also checked in nine screens if a collision between shot / asteroid or ship / asteroid happens.
  (F7) activate FPS display The current FPS rate is displayed in the lower left corner.
  blink delay (value) The items of the main menu and the ship are blinking from time to time. This setting defines how long the program waits between the blink phases. Default is 3.
  sparks (value) When a shot hits an asteroid or if the ship is destroyed, some particles ("sparks") are generated. This setting defines how many. Default is 32.
  (color one) / (color two) The objects in the game are displayed on a monochrome canvas. Here you can change the palette of that canvas. The background will always be the darker color.
  keys This button shows a small window in which you can change the keys of the game (turn left, turn right, accelerate, fire). Not all keys of the keyboard are accessible. You should also avoid multiple assignments.


Notes

The big asteroids won't give you points at all (12 points profit, 12 points loss), but reveal the more worthwile smaller asteroids.

Sometimes, an asteroid can be deviated by destroying it, causing the debris to miss the ship - but likewise the debris can destroy the ship even faster than the big asteroid.

The cannon of the ship reaches 1250° C. (melting point of steel: 1500° C).

The star background that is visible after starting the program was inspired by the movie "Alien".
Ripley deals with the aliens, you clear the path for her escape pod.

Background

The original game ASTEROIDS was developed in 1979 by ATARI as an arcade machine. It uses vector graphics that allow high-resolution graphics instead of the coarse-grained sprites of that time. For each sound effect one electric circuit was developed. In the first version, the player's ship was invulnerable when it was in the score display, so a second version was published. The highscore list had a bug that (under some circumstances) caused the last character of the last player to be cleared.

After publishing in the USA, the arcade owners had to make the coin boxes larger so they could hold the money of the players.

Game description of the arcade cabinet:

Atari Space™ introduces another new space action blockbuster. A new concept that challenges one or two players to compete with themselves or against an opponent for highest scores.
Asteroids is easy to learn but hard to master - a replay natural! The better you players become, the more difficult Asteroids gets. People keep playing to better their previous scores.
Players start with either three or four ships. The object of the game is to maneuver a spacecraft quickly enough to blast and destroy streams of asteroids and computer-controlled enemy saucers - before they can destroy the player's ship.
A direct hit on a large asteroid breaks it in two. A hit on one of these breaks it into two still smaller asteroids, a final hit on a small asteroid destroys it completely.
When the number of asteroids is minimal, a saucer will enter. As the game progresses, more enemy saucers will appear more frequently.
Large saucers shoot in random directions. Small spacecraft will zero in on player's craft. When all asteroids have been destroyed, a new cluster of large asteroids will appear.

Realistic Controls.
Push-button controls are provided for rotation left or right, thrust and fire control. Activation of the "Hyperspace" button causes the spacecraft to go off screen and reappear at random in a different area. Hyperspace is risky however, because of a high probability of destruction upon reentry.
